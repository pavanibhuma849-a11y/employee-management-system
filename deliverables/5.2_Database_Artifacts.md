### Mermaid ER Diagram
```mermaid
erDiagram
    DEPARTMENT ||--o{ EMPLOYEE : "has many"
    EMPLOYEE }|--|{ PROJECT : "works on"

    DEPARTMENT {
        int id PK
        string name
    }

    EMPLOYEE {
        int id PK
        string name
        string role
        decimal salary
        date joining_date
        int department_id FK
    }

    PROJECT {
        int id PK
        string name
        int duration
    }

    EMPLOYEE_PROJECT {
        int employee_id FK
        int project_id FK
    }
```

The diagram illustrates the relationships:
- **Department to Employee**: One-to-Many relationship.
- **Employee to Project**: Many-to-Many relationship managed through a junction table (`employee_project`).
